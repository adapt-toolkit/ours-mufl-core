# core 3.1 — File transfer (`send_file` / `receive_file`)

**Version 3.1 · MIN (additive) · A first-class file-transfer transaction pair,
separate from `send_message`. Files and text are always distinct messages: to
send a file with a caption, an agent sends two transactions.**

## What shipped

`send_file _:($contact, $filename, $mime?, $data, $reply_to?)` mirrors
`send_message` exactly but on its own path: it resolves the contact, stamps a
`wire_id`, and sends the file (`filename` + optional `mime` + `bin` bytes) over
`encrypted_channel` to the new library-routed inbound `::a2a_messaging::receive_file`.
The receiver validates + resolves the sender and hands the file to the app via a
new `on_file_received` storage hook; the sender side fires `on_file_sent`. Both
hooks are injected through `a2a_messaging::init`, identical in shape to the
message hooks. The app owns file storage — the core stores nothing.

## One shared `wire_id` namespace

A file's `wire_id` is stamped from the SAME `_new_id` namespace as a message's, so
`reply_ref_t` (unchanged) works across kinds: a `send_message` can `reply_to` a
file, and a `send_file` can be a reply to a message, out of the box.

## Monitoring

`send_file` / `receive_file` emit the same FORCED monitoring copy as messages,
self-gating on the bound control plane — but the body is **metadata only**:
`[file] <name> (<mime>, <N> B)`. The control plane learns a file moved and its
metadata, never the bytes (`monitoring_copy_t.$body` stays `str`).
When the sender omits the mime type, the `<mime>` field renders empty (e.g. `[file] notes.txt (, 12 B)`).

## Out of scope (YAGNI)

No chunking / streaming / resumable transfer, no content-type validation or dedup,
no attachment-on-message style. One `send_file` = one whole file in one transaction.

## Consumer coordination

`a2a_messaging::init` gains two parameters (`$on_file_received`, `$on_file_sent`).
Consumers (agent packet, web messenger) must add these two hooks to their `init`
call when adopting core 3.1 — the same kind of coordinated, version-gated change as
any prior core signature bump. This repo's QA actor (`tests/test_actor.mu`) is
updated accordingly.
