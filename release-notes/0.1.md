# core 0.1 ‚Äî MIN (additive): `a2a_notifications` ‚Äî protocol-level Notifications-as-a-Service

**Version:** 0.1.0 ¬∑ **Bump:** MIN (additive) ¬∑ One new library defining the
notification-service node surface + a `send_notification` transaction parallel
to `send_message`; no existing wire shape or verification path touched.

## What shipped

- **`a2a_notifications.mm`** ‚Äî one library, two halves:
  - *Client half* (every consumer): `notify_register`, `notify_update_bindings`,
    `notify_rotate_token`, `notify_unregister`, `notify_mark_read`,
    `send_notification`, `export_notify_address` (readonly), inbound
    `confirm_registration`.
  - *Service half* (a notifier packet): `set_vapid_public_key`, inbound
    `register`, `update_bindings`, `rotate_token`, `unregister`,
    `post_notification`, `mark_read`; app hooks for storage/egress
    (`on_notification_posted`, `on_notifications_marked_read`,
    `on_unregistered`; client-side `on_notify_registration`).
- `cap_notifications = "core.notifications"` reserved in `a2a_capabilities.mm`
  (constant only ‚Äî no control verbs, no dispatch change).
- `config.mufl` exports the library; `export_notify_state` /
  `import_notify_state` persistence helpers.
- Contract doc: [`NOTIFICATIONS_CONTRACT.md`](../NOTIFICATIONS_CONTRACT.md).

## New/changed transactions

All new, library-routed `::a2a_notifications::<name>` (no `::actor::` shims).
Nothing existing changed. `post_notification` is the first non-invite inbound
accepted as a **bare signed send** from an unknown packet ‚Äî the shared token
is the sole authorization; validation is version ‚Üí minted-by-me ‚Üí live index
entry ‚Üí `_value_id` byte-equality vs the stored token ‚Üí payload cap (‚â§ 4000).

## Integration TODO for consumers

Nothing required ‚Äî consumers recompile unchanged (new inbound names are never
sent to packets that don't play a half). To adopt: call
`a2a_notifications::init` beside `a2a_messaging::init`, compose
`export_notify_state`/`import_notify_state` into your state wrappers, and (as
a client) drive register ‚Üí subscribe ‚Üí `notify_update_bindings` ‚Üí
`export_notify_address`.

## Security notes

- **No E2E to the device:** the service reads every payload (owner-locked).
  The contract doc carries the loud warning; operators and app authors must
  not put service-hidden secrets in payloads.
- **No anti-spam in v1:** any token holder can post without limit; rotation
  (`notify_rotate_token`) is the recovery path and invalidates the old
  handout atomically (index delete).
- Sender `$from` on a post is recorded but never used for authorization (the
  token is); the recipient surface is encrypted-channel-gated; the VAPID
  private key never enters packet state.
